| 版本 | 支持状态 |
| ---- | -------- |

## 报告漏洞

如果您发现安全漏洞，请 **不要** 通过公开 Issue 报告。

### 报告流程

1. **发送邮件至**: security@example.com
2. **邮件标题**: `[SECURITY] Oceanus Agent - 简短描述`
3. **邮件内容** 请包含:
   - 漏洞类型 (如 SQL 注入、XSS、认证绕过等)
   - 受影响的组件/文件路径
   - 复现步骤
   - 潜在影响评估
   - 建议的修复方案 (如有)

### 响应时间

| 阶段 | 时间目标 |
|------|----------|
| 确认收到 | 24 小时内 |
| 初步评估 | 72 小时内 |
| 修复发布 | 根据严重程度 (Critical: 7 天, High: 14 天, Medium: 30 天) |

### 漏洞披露

- 修复发布后，我们将在 CHANGELOG 和 Release Notes 中公开致谢报告者
- 在修复发布前，请勿公开披露漏洞详情

## 安全最佳实践

### 配置安全

- **API 密钥**: 使用环境变量管理，永远不要硬编码
- **数据库凭证**: 使用 Kubernetes Secrets 或类似机制
- **.env 文件**: 永远不要提交到版本控制

### 依赖安全

- 项目使用 Dependabot 自动检测依赖漏洞
- CI 中包含 Bandit 和 CodeQL 安全扫描
- 建议定期运行 `pip-audit` 检查 Python 依赖

### 运行时安全

- 容器以非 root 用户运行
- 使用最小权限原则配置数据库用户
- 启用 HTTPS (生产环境)
